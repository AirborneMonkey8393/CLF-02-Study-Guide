ğŸ”¹ Benefits of AWS Cloud
-------------------------
â€¢ Agility -- Quickly provision resources and experiment without upfront investment.
â€¢ Elasticity -- Automatically scale resources up or down based on demand.
â€¢ Pay-as-you-go -- Only pay for what you use with no long-term contracts.
â€¢ Global reach -- Deploy applications in multiple geographic regions in minutes.
â€¢ Economies of scale -- Lower costs due to AWS's massive purchasing power.
â€¢ High availability -- Redundant infrastructure minimizes downtime.
â€¢ Stop guessing capacity -- Scale dynamically instead of over-provisioning.
â€¢ CapEx vs OpEx -- Trade upfront capital expenses for variable operational expenses.

ğŸ”¹ AWS Global Infrastructure
-----------------------------
â€¢ Region -- Geographic area with multiple isolated Availability Zones; choose based on latency, compliance, and service availability.
â€¢ Availability Zone (AZ) -- Isolated data center within a region; deploy across multiple AZs for high availability and fault tolerance.
â€¢ Edge Location -- AWS endpoints for CloudFront and Route 53; used for content caching closer to users.
â€¢ Local Zone -- Extension of a region for ultra-low latency applications closer to end users.
â€¢ AWS Outposts -- AWS-managed infrastructure deployed in your on-premises data center for hybrid cloud workloads.

ğŸ”¹ Well-Architected Framework (Six Pillars)
--------------------------------------------
â€¢ Operational Excellence -- Run and monitor systems to deliver business value and continually improve processes.
â€¢ Security -- Protect information, systems, and assets while delivering business value through risk assessments.
â€¢ Reliability -- Ensure workloads perform their intended function correctly and consistently when expected.
â€¢ Performance Efficiency -- Use computing resources efficiently and maintain efficiency as demand changes.
â€¢ Cost Optimization -- Run systems to deliver business value at the lowest price point.
â€¢ Sustainability -- Minimize environmental impact of running cloud workloads.

ğŸ”¹ Cloud Architecture Concepts
-------------------------------
â€¢ High Availability -- System designed to remain operational with redundancy; uses multiple AZs.
â€¢ Fault Tolerance -- System continues operating even when components fail; more robust than HA.
â€¢ Disaster Recovery (DR) -- Strategy to restore systems after catastrophic failure; uses backup regions or Elastic Disaster Recovery.
â€¢ Scalability -- Ability to increase capacity; can be vertical (bigger instance) or horizontal (more instances).
â€¢ Elasticity -- Automatic scaling up and down based on real-time demand; key AWS differentiator.
â€¢ Loose Coupling -- Components operate independently using message queues (SQS) or events (EventBridge) to reduce dependencies.

ğŸ”¹ Cloud Migration Strategies (7 Rs)
-------------------------------------
â€¢ Retire -- Decommission applications no longer needed.
â€¢ Retain -- Keep applications on-premises or delay migration.
â€¢ Relocate -- Move infrastructure to AWS without changes (e.g., VMware Cloud on AWS).
â€¢ Rehost (Lift-and-Shift) -- Move applications to AWS without modifications; fastest migration.
â€¢ Replatform (Lift-and-Reshape) -- Minor optimizations during migration (e.g., RDS instead of self-managed database).
â€¢ Repurchase -- Switch to SaaS or cloud-native alternative.
â€¢ Refactor/Re-architect -- Redesign application to be cloud-native for maximum benefit; most complex.

ğŸ”¹ Shared Responsibility Model
-------------------------------
AWS Responsibility ("Security OF the Cloud"):
  â€¢ Physical security of data centers
  â€¢ Hardware and network infrastructure
  â€¢ Managed service operations (RDS patching, Lambda runtime)
  â€¢ Regions, AZs, edge locations

Customer Responsibility ("Security IN the Cloud"):
  â€¢ Customer data and encryption
  â€¢ IAM user permissions and MFA
  â€¢ OS patching for EC2 instances
  â€¢ Application security and code
  â€¢ Network configuration (security groups, NACLs)
  â€¢ Firewall rules and data encryption in transit

Shared Controls:
  â€¢ Patch management (AWS patches infrastructure; customer patches guest OS)
  â€¢ Configuration management (AWS provides tools; customer configures)
  â€¢ Awareness and training (both responsible)

ğŸ”¹ Identity & Access Management (IAM)
--------------------------------------
â€¢ IAM -- Manage users, groups, roles, and permissions for AWS resources; free service.
â€¢ IAM User -- Long-term credentials for a person or application; use for human access with MFA enabled.
â€¢ IAM Group -- Collection of users with shared permissions; simplifies permission management.
â€¢ IAM Role -- Temporary credentials assumed by users, applications, or services; use for EC2 instances, Lambda, cross-account access.
â€¢ IAM Policy -- JSON document defining permissions (Allow/Deny); attached to users, groups, or roles.
â€¢ Least Privilege Principle -- Grant only the minimum permissions needed for a task.
â€¢ MFA (Multi-Factor Authentication) -- Additional layer of security beyond password; strongly recommended for root user and privileged accounts.
â€¢ Root User -- Account owner with full access; secure with MFA and use only for account setup and billing tasks.
â€¢ AWS STS -- Issues temporary credentials for federated users or role assumptions.
â€¢ IAM Identity Center (formerly SSO) -- Centralized workforce access to multiple AWS accounts and applications.

--------------------
â€¢ AWS KMS (Key Management Service) -- Managed encryption keys integrated with AWS services; customer controls keys.
â€¢ AWS CloudHSM -- Dedicated hardware security module for regulatory compliance requiring single-tenant key storage.
â€¢ AWS Secrets Manager -- Securely store, rotate, and retrieve credentials, API keys, and secrets.
â€¢ AWS Certificate Manager (ACM) -- Provision, manage, and deploy SSL/TLS certificates for free.
â€¢ AWS Shield Standard -- Automatic DDoS protection at no extra cost for all AWS customers.
â€¢ AWS Shield Advanced -- Enhanced DDoS protection with 24/7 response team and cost protection; for mission-critical applications.
â€¢ AWS WAF (Web Application Firewall) -- Filter HTTP/HTTPS traffic to protect against common exploits (SQL injection, XSS).
â€¢ Amazon GuardDuty -- Continuous threat detection using ML to analyze CloudTrail, VPC Flow Logs, and DNS logs.
â€¢ Amazon Inspector -- Automated vulnerability scanning for EC2, containers, and Lambda functions.
â€¢ Amazon Macie -- Discovers and classifies sensitive data (PII) in S3 using machine learning.
â€¢ AWS Security Hub -- Centralized view of security alerts and compliance status across accounts.
â€¢ Amazon Detective -- Investigates security incidents by analyzing log data and relationships.
â€¢ AWS Audit Manager -- Automates evidence collection for compliance audits.
â€¢ AWS Artifact -- Access AWS compliance reports, certifications, and agreements on-demand.
â€¢ AWS Directory Service -- Managed Microsoft Active Directory integration for AWS resources.
â€¢ AWS Firewall Manager -- Centrally manage firewall rules (WAF, Shield) across multiple accounts.
â€¢ AWS RAM (Resource Access Manager) -- Securely share AWS resources across accounts within an organization.

ğŸ”¹ Network Security
-------------------
â€¢ Security Group -- Virtual firewall for EC2 instances at the instance level; stateful (return traffic automatically allowed); default denies all inbound.
â€¢ Network ACL (NACL) -- Firewall at the subnet level; stateless (must configure inbound and outbound rules); default allows all traffic.
â€¢ AWS PrivateLink -- Private connectivity to AWS services and VPCs without internet exposure.
â€¢ AWS VPN -- Encrypted connection between on-premises and AWS over the internet.
â€¢ AWS Direct Connect -- Dedicated private connection between on-premises and AWS; more consistent than VPN.

â€¢ AWS Control Tower -- Automated multi-account setup with governance guardrails based on best practices.
â€¢ AWS Config -- Continuously monitors and records AWS resource configurations; evaluates compliance against rules.
â€¢ AWS CloudTrail -- Logs all AWS API calls for auditing, compliance, and security analysis.
â€¢ AWS License Manager -- Track and manage software licenses (Microsoft, Oracle, SAP) across AWS and on-premises.

ğŸ”¹ Compute Services
-------------------
â€¢ Amazon EC2 -- Virtual servers with full OS control; use when you need custom configurations, legacy apps, or persistent workloads.
  - On-Demand Instances -- Pay by the second with no commitment; for unpredictable workloads.
  - Reserved Instances -- 1- or 3-year commitment for predictable workloads; up to 72% discount.
  - Savings Plans -- Flexible commitment based on $/hour for compute; applies to EC2, Lambda, Fargate.
  - Spot Instances -- Bid on spare capacity for up to 90% discount; can be interrupted; for fault-tolerant batch jobs.
  - Dedicated Instances -- EC2 on hardware dedicated to your account; for compliance.
  - Dedicated Hosts -- Physical server fully dedicated to you; for license requirements (BYOL).

â€¢ AWS Lambda -- Serverless functions triggered by events; pay only for compute time; use for event-driven, short-running tasks instead of always-on servers.
â€¢ AWS Elastic Beanstalk -- PaaS for deploying web apps without managing infrastructure; handles provisioning, load balancing, scaling; you upload code.
â€¢ Amazon Lightsail -- Simplified VPS with bundled resources; for simple websites and small apps; easiest way to launch virtual servers.
â€¢ AWS Outposts -- AWS infrastructure on-premises for hybrid cloud; use when workloads must stay local due to latency or compliance.
â€¢ AWS Batch -- Runs batch computing jobs at scale; automatically provisions optimal compute resources.
â€¢ AWS Fargate -- Serverless container compute; runs containers without managing servers; works with ECS and EKS.

ğŸ”¹ Load Balancing & Auto Scaling
---------------------------------
â€¢ Elastic Load Balancing (ELB) -- Distributes traffic across multiple targets for high availability.
  - Application Load Balancer (ALB) -- Layer 7 (HTTP/HTTPS); routes based on path, host, or headers; for web applications.
  - Network Load Balancer (NLB) -- Layer 4 (TCP/UDP); ultra-high performance for millions of requests per second; for gaming, IoT.
  - Gateway Load Balancer -- Layer 3; routes traffic to third-party virtual appliances (firewalls, IDS).

â€¢ AWS Auto Scaling -- Automatically adjusts capacity based on demand or schedule; maintains availability and optimizes cost.

ğŸ”¹ Storage Services
-------------------
â€¢ Amazon S3 -- Object storage with 11 nines durability; use for backups, static websites, data lakes, media files; not for databases or OS volumes.
  - S3 Standard -- Frequently accessed data.
  - S3 Intelligent-Tiering -- Automatically moves data between access tiers based on usage patterns.
  - S3 Standard-IA (Infrequent Access) -- Lower cost for infrequently accessed data with retrieval fees.
  - S3 One Zone-IA -- Even lower cost for data stored in a single AZ; for non-critical data.
  - S3 Glacier Instant Retrieval -- Archive storage with millisecond retrieval for rarely accessed data.
  - S3 Glacier Flexible Retrieval -- Low-cost archives with retrieval in minutes to hours.
  - S3 Glacier Deep Archive -- Lowest cost for long-term archives; retrieval in 12 hours.
  - S3 Lifecycle Policies -- Automate transitions between storage classes or deletion after a period.
  - S3 Versioning -- Keep multiple versions of objects to protect against accidental deletion.

â€¢ Amazon EBS (Elastic Block Store) -- Persistent block storage volumes for EC2; use for databases and file systems requiring low latency.
â€¢ Amazon EFS (Elastic File System) -- Scalable, shared file system for Linux; multiple EC2 instances can mount simultaneously; use for shared content repositories.
â€¢ Amazon FSx -- Managed file systems:
  - FSx for Windows File Server -- Windows-native shared storage with SMB protocol.
  - FSx for Lustre -- High-performance file system for compute-intensive workloads (HPC, ML).

â€¢ AWS Storage Gateway -- Hybrid storage bridging on-premises and AWS cloud storage.
â€¢ AWS Backup -- Centralized backup service for AWS resources; automates backup schedules and retention policies.
â€¢ AWS Elastic Disaster Recovery (DRS) -- Continuous replication of on-premises or cloud servers to AWS for fast recovery.

ğŸ”¹ Database Services
--------------------
â€¢ Amazon RDS (Relational Database Service) -- Managed relational databases (MySQL, PostgreSQL, SQL Server, Oracle, MariaDB); AWS handles patching, backups, scaling.
â€¢ Amazon Aurora -- AWS-optimized relational database; 5x faster than MySQL, 3x faster than PostgreSQL; compatible with both.
â€¢ Amazon DynamoDB -- Serverless NoSQL key-value/document database; millisecond latency at any scale; for mobile, gaming, IoT.
â€¢ Amazon ElastiCache -- In-memory caching with Redis or Memcached; improves application performance by reducing database load.
â€¢ Amazon Redshift -- Managed data warehouse for analytics and business intelligence; handles petabyte-scale data.
â€¢ Amazon Neptune -- Managed graph database for highly connected data (social networks, fraud detection).
â€¢ Amazon DocumentDB -- MongoDB-compatible document database; managed by AWS.
â€¢ Amazon Keyspaces -- Serverless Cassandra-compatible database for high-throughput applications.
â€¢ Amazon MemoryDB for Redis -- Redis-compatible in-memory database with durability.

ğŸ”¹ Networking & Content Delivery
---------------------------------
â€¢ Amazon VPC (Virtual Private Cloud) -- Isolated virtual network; define IP ranges, subnets, routing, and security.
  - Subnets -- Segments within a VPC; place in specific AZs for high availability.
  - Internet Gateway -- Enables public internet access for VPC resources.
  - NAT Gateway -- Allows private subnet resources to access the internet without being publicly accessible.

  - VPC Flow Logs -- Capture network traffic metadata for monitoring and troubleshooting.

â€¢ Amazon Route 53 -- Scalable DNS service with health checks and routing policies (latency, geo, weighted, failover).
â€¢ Amazon CloudFront -- Global CDN that caches content at edge locations for low latency; use for static/dynamic web content and video streaming.
â€¢ AWS Global Accelerator -- Improves global application availability and performance using AWS network; does not cache content (vs CloudFront).
â€¢ AWS Direct Connect -- Dedicated private network link from on-premises to AWS; more reliable and lower latency than VPN.
â€¢ AWS Transit Gateway -- Central hub to connect multiple VPCs and on-premises networks; simplifies complex network topologies.
â€¢ AWS Site-to-Site VPN -- Encrypted connection between on-premises network and VPC over the internet.
â€¢ AWS Client VPN -- Secure remote access for individual users to AWS and on-premises resources.
â€¢ AWS PrivateLink -- Access AWS services and VPC endpoints privately without internet exposure.
â€¢ Amazon API Gateway -- Create, publish, and manage REST, HTTP, and WebSocket APIs at scale.

ğŸ”¹ Pricing Models
-----------------
â€¢ Free Tier -- Limited free usage of AWS services:
  - Always Free (e.g., Lambda 1M requests/month, DynamoDB 25 GB)
  - 12 Months Free (e.g., EC2 750 hours/month)
  - Trials (e.g., SageMaker)

â€¢ Pay-as-you-go -- Pay only for what you consume with no upfront costs.
â€¢ Save when you commit:
  - Reserved Instances -- 1- or 3-year commitment for EC2; up to 72% discount; best for steady-state workloads.
  - Savings Plans -- Flexible commitment ($/hour) for compute; applies to EC2, Lambda, Fargate; easier than RIs.

â€¢ Pay less by using more -- Volume discounts for services like S3 as usage increases.
â€¢ Pay less when AWS grows -- AWS passes on economies of scale through price reductions.
â€¢ Spot Instances -- Bid on unused capacity for up to 90% savings; can be interrupted; for fault-tolerant batch workloads.

ğŸ”¹ Pricing Factors (Compute)
-----------------------------
â€¢ Instance type -- CPU, memory, storage, and network capacity determine price.
â€¢ Pricing model -- On-Demand, Reserved, Spot, or Savings Plans.
â€¢ Operating system -- Linux is typically cheaper than Windows.
â€¢ Region -- Prices vary by geographic region.
â€¢ Data transfer -- Ingress (inbound) is free; egress (outbound) to internet is charged; transfers between AZs incur small fees.

Q: Which service for real-time streaming data ingestion?
A: Amazon Kinesis Data Streams or Kinesis Data Firehose.

Q: Which service lets you run SQL queries on S3 data without provisioning servers?
A: Amazon Athena.

Q: Which connection provides a private, dedicated link from on-premises to AWS?
A: AWS Direct Connect (not VPN or Internet Gateway).

Q: Which storage is best for a shared file system accessible by multiple Linux EC2 instances?
A: Amazon EFS (not EBS, which is single-instance; not S3, which is object storage).

Q: Which service automatically distributes traffic across EC2 instances in multiple AZs?
A: Elastic Load Balancing (ALB for HTTP/HTTPS, NLB for TCP/UDP).

Q: Which compute option is best for unpredictable, event-driven workloads with sub-second execution?
A: AWS Lambda (serverless, no servers to manage).

Q: Which service provides centralized management of security alerts across multiple accounts?
A: AWS Security Hub.

Q: Which service identifies sensitive data like PII in S3 buckets?
A: Amazon Macie.

Q: Which IAM entity should an EC2 instance use to access other AWS services?
A: IAM Role (not IAM User or long-term credentials).

Q: Which storage class is best for data accessed once per year?
A: S3 Glacier Deep Archive (lowest cost, 12-hour retrieval).

Q: Which service helps you migrate a 500 TB on-premises data center to AWS when bandwidth is limited?
A: AWS Snowball Edge or Snowmobile (not DMS or internet transfer).

Q: Which service provides automated threat detection using machine learning?
A: Amazon GuardDuty.

Q: Which AWS service records all API calls for auditing?
A: AWS CloudTrail.

Q: Which tool provides real-time recommendations for cost optimization and security?
A: AWS Trusted Advisor (more checks available with Business/Enterprise support).

Q: Which service allows you to deploy infrastructure using code templates?
A: AWS CloudFormation (IaC).

Q: In the Shared Responsibility Model, who is responsible for patching the guest OS on EC2?
A: Customer (AWS patches underlying hypervisor and hardware).

Q: Which database is best for high-scale, low-latency key-value workloads?
A: Amazon DynamoDB.

Q: Which service provides a dedicated technical advisor for proactive guidance?
A: Technical Account Manager (TAM) with Enterprise Support.

Q: Which service enables you to run containers without managing servers?
A: AWS Fargate (not EC2 or ECS alone).

Q: Which pricing model offers the highest discount for interruptible batch workloads?
A: Spot Instances (up to 90% discount).

Q: Which service is best for caching database query results to improve performance?
A: Amazon ElastiCache.
